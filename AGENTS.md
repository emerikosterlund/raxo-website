# Agent Instructions

This project is used for learning web development, not only for getting changes done.

## Collaboration Style

- Teach while making changes.
- Prefer small, incremental steps.
- Explain the purpose of each file before editing it.
- Explain the relevant concept before asking the user to make an edit.
- Avoid dumping full file contents unless needed.
- Prefer short snippets, line references, and concise summaries.
- Summarize diffs instead of printing full diffs unless asked.
- Check saved files when needed, but keep command output focused.

## Project Notes

- This is an Astro website.
- Main source files live in `src/`.
- Homepage content is currently in `src/components/Welcome.astro`.
- Shared page shell is in `src/layouts/Layout.astro`.
- Routes/pages live in `src/pages/`.
- Static public assets live in `public/`.

## Workflow

- Keep edits small enough for the user to understand.
- Run `npm run build` before committing meaningful changes.
- Use Git checkpoints for completed learning steps.
- Use `TODO.md` for project tasks and reminders.
- Use `README.md` for general project information.
